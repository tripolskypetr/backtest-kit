# TODO: Локализация фронтенда (packages/front/modules/frontend)

Задача: каждый файл `src/**/*.ts*` прочитан **целиком**, все пользовательские тексты обёрнуты в `t()`, переводы добавлены в `src/i18n/locale/locale.en.ts` и `locale.ru.ts`.

## Правила

- Импорт: `import { t } from "<относительный путь>/i18n";` (см. существующие использования, напр. `src/components/StatusInfo.tsx`).
- Ключ = английская фраза как есть. В `locale.en.ts` значение = ключ, в `locale.ru.ts` — русский перевод.
- **Оборачивать**: `title`, `placeholder`, `label`, `description`, тексты кнопок/тултипов, тексты в `compute` (напр. `"Not specified"`, `"N/A"`, `"Backtest"`, `"Live"`), сообщения тостов/alert/confirm, JSX-текст в собственных компонентах (jsx-фабрика — только страховка для сторонних библиотек), заголовки секций форм (`FieldType.Typography` → `placeholder`).
- **Не оборачивать**: `name`, `type`, ключи объектов, форматы dayjs, роуты, технические значения, console.*, идентификаторы, пустые строки, символы (`$`, `%`, `+`).
- **USDT → RUB**: везде, где в UI встречается `USDT`, оборачивать в `t("USDT")`; в `locale.ru.ts`: `"USDT": "RUB"` (Мосбиржа). Строки, содержащие USDT внутри, в ru-переводе тоже используют RUB.
- Шаблоны форм (`src/assets/*_fields.tsx` и подобные) — это JSON-структуры react-declarative: греп по JSX их не находит, читать целиком.
- Файл без пользовательских текстов помечается выполненным после полного прочтения (пометка `нет текстов`).
- Дубли между модулями (напр. `hooks/*/components/StockChart`) — копипаста намеренная, править каждую копию отдельно.
