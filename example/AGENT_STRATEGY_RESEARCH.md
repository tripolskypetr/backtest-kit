# BTCUSDT February 2026 — Agent Strategy Research

## Итог

**+36.74% за февраль 2026** при max equity drawdown **-2.41%**.

---

## Архитектура системы

Три независимых слоя:

```
[AI Agent]  →  каждые 8h выдаёт BUY / SELL / WAIT
     ↓
[Pine файл]  →  на 1m свечах находит локальный экстремум для входа
     ↓
[Стратегия]  →  держит позицию, выходит по трейлингу или SL
```

---

## Слой 1 — AI Agent (research)

### Что делает

`research(symbol, when)` — запускается каждые 8 часов (00:00, 08:00, 16:00 UTC).

Агент ищет в интернете острые событийные триггеры последних 4-12 часов и возвращает:

```ts
{ signal: "BUY" | "SELL" | "WAIT", reasoning: string }
```

### Приоритеты поиска (от важного к менее важному)

1. **Острые события**: взлом биржи, решение регулятора (SEC/CFTC/DOJ), flash crash, твит Трампа, решение по ETF
2. **Макро-девиации**: неожиданное решение ФРС, CPI выше/ниже ожиданий, резкое движение DXY
3. **Объёмные аномалии**: unusual volume spike, sudden price move
4. **Прогнозы аналитиков**: как наименее значимый источник

### Правила агента

- Только события последних 4-12 часов — никакой инертной аналитики
- Если дату источника нельзя определить явно — не использовать
- Несколько запросов поиска для подтверждения из разных источников
- Если картина противоречивая или данных мало — **WAIT**

### Статистика за февраль 2026 (84 блока)

| Сигнал | Кол-во | % |
|--------|--------|---|
| SELL   | 45     | 54% |
| WAIT   | 34     | 40% |
| BUY    | 5      | 6% |

Доминирующий bias: **медвежий** — февраль 2026 был месяцем краша BTC $79k → $60k.

### Ключевое ограничение агента

Агент **опаздывает на 8h** — сигнал формируется по событиям прошлого блока. Поэтому:
- SELL на абсолютном дне (Feb 6 00:00, $60k) — системная ошибка агента, не баг
- SL в стратегии защищает именно от этого сценария

## Слой 2 — Pine файлы (локальный вход)

### Зачем нужны

Агент говорит направление, но входить сразу по close блока неэффективно. Анализ 1m свечей всех 41 SELL блока февраля показал:

- В **40 из 41** SELL блоков цена сначала спайкует вверх в первые 60 минут перед основным падением
- Средний MaxUp перед падением: **+0.9%**
- Вход на локальном хае вместо open снижает adverse drawdown в 40/41 случаях

Для BUY: в **5 из 5** блоков цена сначала проседает в первые 60 минут перед ростом.

### position_short.pine

**Когда запускается:** агент вернул SELL.

**Логика входа:**
```
windowHigh = highest(high, 60)   // хай последних 60 баров
localTopConfirmed = high[1] >= windowHigh AND close < high[1]
// предыдущий бар сделал хай 60-барного окна И текущий бар закрылся ниже
entryPrice = high[1]             // цена входа = хай бара, сформировавшего вершину
```

**Выходы:** `position=1` пока держим SHORT, `position=0` когда выходим. TP/SL не в pine — только EntryPrice.

**Stateless:** нет `var`, нет персистентного состояния. Каждый бар пересчитывается с нуля.

### position_long.pine

**Когда запускается:** агент вернул BUY.

**Логика входа:**
```
windowLow = lowest(low, 60)
localBottomConfirmed = low[1] <= windowLow AND close > low[1]
entryPrice = low[1]
```

**Выходы:** `position=1` пока держим LONG, `position=0` когда выходим.

### Тайминг запуска pine

Pine запускается на **1m свечах** строго после сигнала агента. Пока агент возвращает один и тот же сигнал (например SELL несколько 8h блоков подряд) — pine продолжает работать, позиция **не закрывается**. Это ключевое: несколько consecutive SELL блоков = один длинный тренд = одна позиция.

## Слой 3 — Стратегия (управление позицией)

### Открытие позиции

```
if agent.signal == "SELL":
    запустить position_short.pine на 1m барах
    когда pine.position переходит 0 → 1: открыть SHORT по pine.EntryPrice

if agent.signal == "BUY":
    запустить position_long.pine на 1m барах
    когда pine.position переходит 0 → 1: открыть LONG по pine.EntryPrice

if agent.signal == "WAIT":
    ничего не делать
```

### Удержание позиции

Позиция держится пока `pine.position == 1`. Условий для досрочного выхода два:

**1. Трейлинг-выход (основной):**
```
if getPositionHighestProfitMinutes(symbol) >= 30:
    закрыть позицию
```

Логика: прошло 30 минут с момента пика прибыли — цена развернулась, движение завершено.

Обоснование порога 30 минут (из backtesta всех SELL блоков):

| Порог | Total PnL | vs baseline |
|-------|-----------|-------------|
| baseline (без трейлинга) | +28.30% | — |
| **30 min** | **+36.74%** | **+8.44%** |
| 60 min | +31.19% | +2.89% |
| 90 min | +30.08% | +1.78% |
| 360 min | +26.96% | -1.34% |

**2. Stop-Loss (защита от аномалии):**
```
SL = entryPrice * 1.03   // для SHORT: +3% от входа
SL = entryPrice * 0.97   // для LONG:  -3% от входа
```

Обоснование 3%: максимальный adverse move после входа на локальном экстремуме за весь февраль — **-3.12%** (Feb 6, SELL прямо на абсолютном дне $60k, V-разворот +6%). Это единственный случай когда агент принципиально ошибся с направлением.

**3. Take-Profit:**
```
TP = Infinity  (недостижимый, например +50%)
```

TP не используется — выход только через трейлинг или SL.

### Закрытие при смене сигнала

Когда агент переходит из SELL в WAIT или BUY — pine перестаёт вызываться, `pine.position` возвращает 0. Стратегия закрывает позицию.

### Повторный вход после трейлинг-выхода

**Не входить.** Анализ всех 13 случаев трейлинг-выхода показал:
- После выхода цена ни разу не обновила минимум (`newLow=false` везде)
- После выхода цена всегда сначала отскакивает вверх (bounceFirst: +0.34% до +6.29%)
- Повторный вход в тот же блок: 6 плюсов / 5 минусов, матожидание ≈ 0

Ждать следующего сигнала агента.

## Ключевые выводы

### Почему это работает

1. **Агент правильно определяет тренд.** 58.5% точность по направлению — но при 7-блочных трендах (Feb 4-6, Feb 21-23) движение составляет 15-20%, что компенсирует все убытки месяца.

2. **Pine улучшает вход.** Вход на локальном хае первых 60 минут работает в 40/41 SELL блоках. Без pine — вход по open, adverse drawdown выше.

3. **Трейлинг 30 минут оптимален.** Не срезает прибыль раньше времени на длинных трендах, но защищает от отдачи на коротких блоках. +8.44% vs baseline без трейлинга.

4. **Consecutive SELL блоки = один тренд.** Закрывать позицию на границе каждого 8h блока — ошибка. Агент "видит" тренд несколько блоков подряд, и позиция должна жить всё это время.

5. **BUY сигналы слабые.** Только 5 за месяц, одна значимая сделка (Feb 25, short squeeze +3.92%). Не торговать BUY агрессивно.

### Риски

- **Разворот тренда при SELL-цепочке**: Feb 6 00:00 — агент дал SELL прямо на дне $60k. SL 3% защищает от этого.
- **Агент опаздывает на 8h**: сигнал формируется по событиям уже закрытого блока. Нельзя ждать подтверждения — нужно входить на следующем блоке.
- **Thin liquidity на 1m**: точки входа pine теоретические, в реальности slippage на локальных хаях может быть выше.

## Параметры для стратегии

```ts
// pine файлы
// content/feb_2026.strategy/math/position_short.pine  — для SELL
// content/feb_2026.strategy/math/position_long.pine   — для BUY
// timeframe: 1m, windowBars: 60

// агент
interval: "8h"
cacheKey: "research_source"

// управление позицией
SL_SHORT = entryPrice * 1.03   // +3%
SL_LONG  = entryPrice * 0.97   // -3%
TP = Infinity                   // выход только по трейлингу

// трейлинг-выход
if (await getPositionHighestProfitMinutes(symbol) >= 30) {
    // закрыть позицию
}

// повторный вход после трейлинга — нет
```
